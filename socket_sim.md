# Memory Rubber Socket Contact Simulator (메모리 러버 소켓 접촉 시뮬레이터)

이 시뮬레이터는 **메모리 러버 소켓(Memory Rubber Socket)** 및 주변 기계 구조물들(PCB, Socket Guide, Insert, Heat Sink, Device) 간의 물리적 배치와 압축 상태를 계산하고 실시간으로 시각화하는 **Tkinter 기반 GUI 애플리케이션**입니다.

하드웨어 설계 단계에서 치수 공차 및 가이드 높이 설계 미스로 인해 발생할 수 있는 **과압축(Over-compressed)으로 인한 디바이스 파손**이나 **접촉 불량(No Contact)으로 인한 작동 불량**을 사전에 방지하도록 설계되었습니다.

---

## 📌 주요 기능
1. **실시간 파라미터 제어**: 슬라이더를 사용하여 각 구조물의 치수를 실시간으로 조절할 수 있습니다.
2. **0.05 mm 해상도 정밀 스냅**: 실제 가공 공차 환경을 반영하여 모든 입력 파라미터 값은 **0.05 mm 단위**로 정밀하게 반올림(Snap) 처리됩니다.
3. **4가지 상태 자동 판정**: 소켓의 압축 특성과 물리적 한계를 연산하여 안전성을 직관적으로 판정합니다.
4. **2D 기하 구조 실시간 시각화**: 기계 구조물의 조립 단면을 컬러 그래픽으로 캔버스에 실시간 드로잉합니다.
5. **시각적 오류 및 마커 표시**: 과압축으로 인한 디바이스 변형량(Crushed), 접촉 불량 시 갭(Gap) 크기, 주요 물리적 참조선들을 동적으로 렌더링합니다.

---

## 📐 핵심 계산 수식 및 매커니즘

시뮬레이터는 다음 3가지 핵심 누적 높이 변수를 기준으로 연산을 수행합니다.

| 기호 | 변수명 | 수식 | 설명 |
| :---: | :--- | :--- | :--- |
| **`Val 1`** | **최소 허용치 (Hard Stop + Device)** | $S_{hard} + D_h$ | 소켓이 물리적으로 더 이상 찌그러질 수 없는 한계(`S_hard`)까지 압축되었을 때의 전체 높이입니다. 구조물이 이 값보다 낮아지면 디바이스가 짓눌려 파손됩니다. |
| **`Val 2`** | **구조물 압축 목표치** | $G_h + I_h + h_g$ | 가이드(`G_h`)와 인서트(`I_h`), 그리고 방열판 홈 깊이(`h_g`)의 합으로, 방열판이 완전히 덮였을 때 물리적으로 고정되는 총 공간 높이입니다. |
| **`Stack`** | **초기 적층 높이** | $S_{max} + D_h$ | 소켓이 전혀 압축되지 않은 자연 상태일 때의 소켓 높이와 디바이스 높이의 합입니다. |

## 🎛️ 입력 파라미터 (Inputs)

| 파라미터명 (코드 변수) | 범위 (mm) | 설명 | 색상 (시각화) |
| :--- | :---: | :--- | :---: |
| **Socket 최대 높이 (`s_max`)** | 0.1 ~ 1.5 | 압축되지 않은 소켓 고유의 최대 높이 | 노란색 (`#FFCA28`) |
| **Socket Hard Stop (`s_hard`)** | 0.2 ~ 1.3 | 소켓이 파손 없이 버틸 수 있는 최소 두께 한계 | (소켓 하단 참조선) |
| **Device 높이 (`d_h`)** | 8.0 ~ 12.0 | 소켓 위에 탑재되는 반도체 패키지 디바이스의 높이 | 하늘색 (`#42A5F5`) |
| **Socket Guide 높이 (`g_h`)** | 3.0 ~ 6.0 | PCB에 고정되어 소켓을 감싸는 가이드의 높이 | 검은색 (`#424242`) |
| **Insert 높이 (`i_h`)** | 3.0 ~ 6.0 | 가이드 위에 조립되어 누름판을 지지하는 인서트의 높이 | 민트색 (`#26A69A`) |
| **Heatsink Device 홈 깊이 (`h_g`)** | 0.0 ~ 3.0 | 방열판 하단에 파여 있는 디바이스 안착용 홈의 깊이 | 보라색 (`#AB47BC`) |

## 🚦 시뮬레이션 결과 판정 기준

시뮬레이터는 계산된 높이를 비교하여 **4가지 상태**를 도출합니다.

### 1. ❌ Fail (Over-compressed) - 과압축 상태
* **조건**: `Val 2 < Val 1 - 1e-5`
* **설명**: 방열판이 닫히는 목표 높이(`Val 2`)가 소켓의 물리적 한계 압축 높이(`Val 1`)보다 낮습니다. 이 상태에서는 방열판 힘에 의해 디바이스 패키지 자체가 찌그러지며 손상됩니다.
* **출력 메커니즘**:
  * 소켓 압축 길이: 최대 압축량 ($S_{max} - S_{hard}$) 표시
  * 디바이스 간섭량 (Crushed): 부족한 수치 ($Val 1 - Val 2$) 표시
  * 시각화: 디바이스 색상이 빨간색 (`#E53935`)으로 변하며 **"DEVICE CRUSHED"** 경고 텍스트 및 간섭 마커가 붉은색 화살표로 그려집니다.

### 2. ⚠️ Fail (No Contact) - 접촉 불량 상태
* **조건**: `Val 2 >= Stack - 1e-5`
* **설명**: 닫히는 높이(`Val 2`)가 초기 디바이스 상단 높이(`Stack`)보다 크거나 같습니다. 방열판이 소켓의 자연 높이에 닿지 못하므로, 디바이스와 소켓 사이에 물리적인 갭(Gap)이 존재하여 전기적 접촉이 전혀 일어나지 않습니다.
* **출력 메커니즘**:
  * 소켓 압축 길이: `0.00 mm` (미압축)
  * 모자란 길이 (Gap): 떨어진 갭 두께 ($Val 2 - Stack$) 표시
  * 시각화: 방열판과 디바이스 사이에 주황색 (`#F57C00`) 양방향 화살표와 갭 수치가 표시됩니다.

### 3. ✅ Pass - 최적 접촉 상태
* **조건**: `abs(Val 1 - Val 2) < 1e-5` (즉, `Val 1 == Val 2`)
* **설명**: 방열판의 안착 완료점과 소켓의 하드 스톱 도달점이 완벽하게 일치하는 가장 이상적이고 정밀한 기계적 매칭 상태입니다.
* **출력 메커니즘**:
  * 소켓 압축 길이: 완벽 조화량 ($Stack - Val 2$) 표시
  * 결과 텍스트: 초록색 (`#388E3C`) **"결과: Pass"** 출력

### 4. ⚡ 조건부 Pass (Conditional Pass)
* **조건**: `Val 1 < Val 2 < Stack`
* **설명**: 소켓이 하드 스톱까지 도달하지는 않았지만, 충분한 압축 구간에 진입하여 완충력을 가지며 안전하게 안착된 기계적으로 안정적인 접촉 구간입니다.
* **출력 메커니즘**:
  * 소켓 압축 길이: $Stack - Val 2$ 표시
  * 결과 텍스트: 황색 (`#F9A825`) **"조건부 Pass (X.XXmm 부족)"** 출력 (하드 스톱 도달까지 남아있는 마진을 알려줍니다.)

## 🚀 실행 방법

본 시뮬레이터는 Python 3 환경에서 가상환경 `uv`를 이용하여 실행할 수 있습니다.

### 1. 의존성 패키지 설치
이 프로그램은 Python 표준 라이브러리인 `tkinter`만을 사용하여 추가적인 무거운 그래픽 엔진 패키지 없이 경량으로 구동됩니다.

### 2. 실행 명령
터미널을 통해 프로젝트 디렉토리 경로로 이동 후 아래 명령을 통해 실행합니다.
```bash
python socket_sim.py
```
*(또는 가상환경 실행)*
```bash
.venv\Scripts\python socket_sim.py
```
