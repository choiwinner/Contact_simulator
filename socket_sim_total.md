# Integrated Memory Socket Contact Simulator (통합 메모리 소켓 접촉 시뮬레이터)

이 시뮬레이터는 **Latch (Dial) 가압 방식**과 **HeatSink 스프링 볼트 체결 방식** 두 가지 소켓 접촉 시나리오를 하나로 통합하여, 메모리 러버 소켓(Memory Rubber Socket) 및 주변 기계 구조물들 간의 물리적 조립 상태와 압축 거동을 연산하고 실시간으로 시각화하는 **Tkinter 기반 GUI 애플리케이션**입니다.

두 가지 다른 가압 메커니즘을 실시간으로 전환하며 비교 분석할 수 있어, 치수 공차 설계 미스로 인한 **디바이스 파손(Over-compression)** 또는 **접촉 불량(No Contact)** 문제를 사전에 방지하는 설계 검증 툴로 활용됩니다.

---

## 📌 주요 특징 및 기능

1. **실시간 모드 전환 (Latch ↔ HeatSink)**
   - 좌측 제어 패널 상단의 모드 버튼을 클릭하여 시뮬레이터 방식을 즉시 전환할 수 있습니다. 
   - 모드 전환 시, 입력 파라미터 슬라이더 목록, 연산 결과 조건식, 캔버스 도면 및 범례가 실시간으로 재배치됩니다.

2. **패키지 레시피(Recipes) 연동 기능**
   - **Rubber Socket**: HBM3E, HBM4, HBM4E 규격 또는 사용자 입력을 선택하여 소켓의 최대 높이(`S_max`)와 하드 스톱 한계치(`S_hard`)를 원클릭으로 설정할 수 있습니다.
   - **Dial Offset (Latch 전용)**: HBM3E Latch (0.40mm), HBM4 Latch (0.50mm) 또는 사용자 입력을 받아 다이얼의 오프셋 치수를 자동 설정합니다.

3. **0.05 mm 해상도 정밀 스냅**
   - 가공 공차 환경을 반영하여 슬라이더로 조절하는 모든 입력 치수 값은 **0.05 mm 단위**로 정밀하게 반올림(Snap) 처리되어 연산에 반영됩니다.

4. **동적 2D 기하 구조 시각화**
   - **Latch 모드**: ㄷ자형 Metal Lid 내부에서 회전하는 다이얼 노브, 회전 각도에 동기화되는 원형 Dial Gauge, 다이얼 나사축을 디테일하게 표현합니다.
   - **HeatSink 모드**: 23개의 조밀한 세로 방열 핀(Fins), 4쌍의 측면 가로 핀, 실제 조임 상태를 묘사하는 스프링 장착 볼트(Spring-loaded Screws) 및 나사선 체결 홈(Thread hole)을 고해상도로 렌더링합니다.

## 📐 핵심 계산 수식 및 매커니즘

시뮬레이터는 선택한 가압 모드에 따라 서로 다른 높이 누적 변수를 연산합니다.

### 1. 공통 기준 변수
* **`Val 1` (최소 허용치 - Hard Stop + Device)**: $S_{hard} + D_h$
  - 소켓이 물리적으로 더 이상 찌그러질 수 없는 한계(`S_hard`)까지 압축되었을 때의 디바이스를 포함한 최소 필요 공간입니다. 내부 유격 공간이 이 값보다 낮아지면 디바이스에 압축 압력이 초과 인가되어 파손됩니다.
* **`Stack` (초기 적층 높이)**: $S_{max} + D_h$
  - 소켓이 압축되지 않은 자연 상태일 때의 소켓 높이와 디바이스 높이의 합입니다.

### 2. 모드별 유격 공간 변수
* **Latch 모드 - 실제 내부 공간 (`Space`)**: $Dial + Dial\ Offset$
  - 다이얼 나사의 조임 설정값과 다이얼 오프셋의 합으로, 디바이스가 위치할 수 있게 가압 블록(Pusher)이 허용하는 실제 높이 공간입니다.
* **HeatSink 모드 - 구조물 압축 목표치 (`Val 2`)**: $G_h + I_h + h_g$
  - 소켓 가이드(`G_h`)와 인서트(`I_h`), 그리고 방열판 하단 홈 깊이(`h_g`)의 합으로, 방열판 볼트가 완전히 체결되었을 때 디바이스가 수용되는 실제 높이 공간입니다.

## 🚦 시뮬레이션 결과 판정 기준

시뮬레이터는 공간 변수(`Space` 또는 `Val 2`)를 최소 허용치(`Val 1`) 및 초기 적층 높이(`Stack`)와 비교하여 **4가지 상태**를 판정하고 화면에 실시간으로 표시합니다.

| 판정 결과 | 조건 (Latch) | 조건 (HeatSink) | 설명 및 그래픽 연출 |
| :---: | :--- | :--- | :--- |
| **❌ Fail<br>(Over-compressed)** | `Space < Val 1` | `Val 2 < Val 1` | **과압축 상태**: 가압 공간이 소켓 하드 스톱 한계치보다 낮아 디바이스 패키지가 파손됩니다. 캔버스 상의 디바이스가 빨간색으로 변하고 **"DEVICE CRUSHED"** 경고 문구 및 간섭량(Crushed) 화살표 마커가 붉은색으로 강조됩니다. |
| **⚠️ Fail<br>(No Contact)** | `Space >= Stack` | `Val 2 >= Stack` | **접촉 불량 상태**: 가압판이 소켓의 자연 높이에 도달하지 못해 디바이스와 소켓 핀 간의 접촉이 발생하지 않습니다. 캔버스 상에 주황색 점선 및 갭(Gap) 크기를 나타내는 양방향 화살표가 표시됩니다. |
| **✅ Pass** | `Space == Val 1` | `Val 2 == Val 1` | **최적 접촉 상태**: 가압 완료 지점이 소켓의 하드 스톱 도달점과 정밀하게 일치하여 접촉이 완벽하게 이루어진 이상적인 설계 상태입니다. 초록색으로 표시됩니다. |
| **⚡ 조건부 Pass** | `Val 1 < Space < Stack` | `Val 1 < Val 2 < Stack` | **안전 접촉 상태**: 소켓이 하드 스톱까지 완전히 눌리지는 않았으나, 충분히 안정적인 압축 가압 구간에 위치하여 정상 작동하는 기계적 매칭 상태입니다. 황색으로 표시됩니다. |

## 🎛️ 입력 파라미터 (Inputs) 목록

| 모드 | 파라미터명 (변수명) | 범위 (mm) | 설명 |
| :---: | :--- | :---: | :--- |
| **공통** | **Socket 최대 높이 (`s_max`)** | 0.10 ~ 1.50 | 압축되지 않은 소켓 고유의 높이 (레시피 선택 시 고정) |
| **공통** | **Socket Hard Stop (`s_hard`)** | 0.20 ~ 1.30 | 소켓이 파손 없이 견딜 수 있는 최소 한계 두께 (레시피 선택 시 고정) |
| **공통** | **Device 높이 (`d_h`)** | 8.00 ~ 12.00 | 테스트 대상 메모리 패키지 디바이스의 높이 |
| **Latch** | **Dial 값 (`dial_val`)** | 8.00 ~ 11.00 | 다이얼 게이지 조임 값 |
| **Latch** | **Dial Offset (`dial_offset`)** | 0.00 ~ 2.00 | 다이얼 노브의 조임 기준 치수 오프셋 |
| **HeatSink** | **Socket Guide 높이 (`g_h`)** | 3.00 ~ 6.00 | PCB 위에 설치되는 소켓 가이드의 외부 높이 |
| **HeatSink** | **Insert 높이 (`i_h`)** | 3.00 ~ 6.00 | 소켓 가이드 상단에 위치하는 인서트 스페이서의 높이 |
| **HeatSink** | **Heatsink Device 홈 깊이 (`h_g`)** | 0.00 ~ 3.00 | 방열판 하부에 파여진 디바이스 수용 홈 깊이 |

## 🚀 실행 방법

시뮬레이터는 Python 3 환경에서 추가적인 그래픽 라이브러리 설치 없이 기본 내장된 `tkinter` 모듈을 사용하여 빠르고 가볍게 구동됩니다.

### 1. 가상환경 진입 (선택 사항)
`uv` 패키지 매니저로 구성된 환경인 경우, 프로젝트 경로에서 가상환경을 활성화합니다.
```powershell
.venv\Scripts\activate
```

### 2. 통합 시뮬레이터 실행
터미널 창에 아래 명령어를 입력하여 프로그램을 실행합니다.
```bash
python socket_sim_total.py
```
*(또는 가상환경 실행)*
```bash
.venv\Scripts\python socket_sim_total.py
```
